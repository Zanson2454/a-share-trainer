# 选股实现逻辑

## 股票池概念

系统中有两层股票池：

### 全市场股票池（选股范围）

**A 股全部 ~5500 只个股**，自动排除 ST、退市、N（新股）、C（次新股）。

数据来源：
- Sina 实时行情定期同步到本地缓存（`~/.cache/stock_data/stock_basic.json`）
- 缓存按 Tushare 的 `stock_basic` 接口设计，含 `list_status`（L/D/P）、`list_date`、`delist_date`
- 支持增量更新：新上市、退市、改名均可识别
- 缓存过期时间 1 天，过期后自动同步

所有选股 API 均以此为基准池，再根据不同的入口场景施加过滤条件。

### 个人股票池（入选结果）

**经过五维模型评分后，总分 ≥ 70 分** 的股票进入个人观察池。这是选股的**结果**，不是选股的**范围**。

---

## 各场景的选股范围差异

| 场景 | 刷选范围 | 说明 |
|------|---------|------|
| **结构化选股** `GET /api/screening` | 涨幅榜前20中取前5 | 从全市场实时行情中取涨幅最大的20只，按市值排序取前5只评分 |
| **自然语言选股** `POST /api/screening/natural` | 全市场 ~5500只 → 关键词过滤 | 按行业关键词、风格、估值条件过滤后，截取市值前24只评分 |
| **用户指定代码** `GET /api/screening?codes=xxx` | 用户指定的代码 | 直接对用户传入的代码列表评分 |
| **CLI 命令** `/选股` | 全市场 ~5500只 → 市值/PE过滤 | 按市值≥50亿、PE>0且≤200 过滤后评分 |

结构化选股的数据获取优先级：
1. 用户指定的代码列表（精确匹配）
2. 全市场实时涨幅榜（AKShare `stock_zh_a_spot_em`）
3. 兜底方案：硬编码 5 只蓝筹（仅当 AKShare 完全不可用时触发，防止服务崩溃）

所有场景均自动排除 ST、退市、N（新股）、C（次新股）标记的股票。

## 五维评分模型

总分 **100 分**，入池阈值 **≥ 70 分**。

### 1. 大盘环境 — 0~20 分

基于上证指数（000001）的 MA20/MA60 判定市场趋势：

| 条件 | 分类 | 加减分 |
|------|------|--------|
| 价格 > MA20 > MA60 | 进攻 | +5 |
| 价格在 MA20 与 MA60 之间 | 震荡 | 不变 |
| 价格 < MA20 且 < MA60 | 防守 | -3 |
| 指数涨跌幅 > +1% | — | +3 |
| 指数涨跌幅 < -1% | — | -3 |
| 成交量 > 20日均量 × 1.2 | — | +2 |

得分限制在 [0, 20] 区间。大盘环境评分对所有候选股票相同。

---

### 2. 政策/热点 — 0~20 分

调用 `ak.stock_board_industry_name_em()` 获取行业板块涨跌排行：

- 板块涨幅 > 3% → **主线**
- 板块涨幅 > 1.5% → **支线**
- 其余 → **一日游**

| 情况 | 得分 |
|------|------|
| 成功获取板块数据 | **15 分** |
| 板块数据不可用 | **5 分** |

此维度得分对所有候选股票相同，不按个股行业匹配。

---

### 3. 基本面 — 0~25 分

基准分 **12.5 分**，根据以下指标调整：

| 指标 | 条件 | 加减分 | 含义 |
|------|------|--------|------|
| PE（市盈率） | 0 < PE < 30 | +3 | 估值合理 |
| PE | PE > 100 | -3 | 估值偏高 |
| ROE（净资产收益率） | ROE > 15% | +4 | 盈利能力优秀 |
| ROE | 8% ≤ ROE ≤ 15% | +2 | 盈利能力良好 |
| 利润增速 | 增速 > 20% | +3 | 高增长 |
| 负债率 | < 50% | +3 | 低风险 |
| 负债率 | > 80% | -3 | 高杠杆 |

得分限制在 [0, 25] 区间。

财务数据来源于三源合并（优先级递减）：
1. **AKShare** `stock_financial_analysis_indicator` — 完整财务指标
2. **Sina 实时接口** — PE、PB、市值（快速但浅层）
3. **Tushare** `get_financial_indicators` — ROE、负债率、增速（深层但需 API token）

---

### 4. 技术面 — 0~25 分

基准分 **12.5 分**，基于均线排列和成交量：

| 条件 | 加减分 | 含义 |
|------|--------|------|
| 价格 > MA5 > MA20 | +5 | 多头排列 |
| 价格 < MA5 < MA20 | -3 | 空头排列 |
| 价格 > MA60 | +3 | 站上60日线 |
| 当日成交量 > 20日均量 × 1.5 | +2 | 放量 |

得分限制在 [0, 25] 区间。

---

### 5. 风险控制 — 0~10 分

固定为 **8 分**，未做动态计算。

---

## 评分流程

```
候选代码列表
     │
     ▼
┌─────────────────┐
│ 获取K线数据       │  AKShareClient.get_daily_kline(code)
│ 计算MA5/MA20/MA60│
└────────┬────────┘
         ▼
┌─────────────────┐
│ 获取财务数据       │  三源合并（AKShare → Sina → Tushare）
│ PE/PB/ROE/增速    │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 获取板块数据       │  AKShareClient.get_hot_sectors()
│ （大盘/政策维度）   │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 五维分别打分       │
│ 汇总总分          │
└────────┬────────┘
         ▼
┌─────────────────┐
│ total >= 70 ?    │─── 是 ──→ 纳入个人股票池
│                  │─── 否 ──→ 过滤掉
└─────────────────┘
```

结构化选股最多评分 **5 只**，自然语言选股最多评分 **8 只**。

## 自然语言选股

### 关键词解析

系统将用户输入的自然语言拆解为四个维度：

#### 行业关键词（18个行业）

| 行业 | 匹配的品牌/股票名模式 |
|------|---------------------|
| 白酒 | 茅台、五粮、老窖、汾酒、古井、洋河等 |
| 消费 | 伊利、蒙牛、海天、金龙鱼、双汇等 |
| 新能源 | 宁德、比亚迪、阳光电源、隆基、通威等 |
| 电池 | 宁德、亿纬、国轩、欣旺达、德赛等 |
| 光伏 | 隆基、通威、阳光、晶澳、天合、中环等 |
| 医药 | 恒瑞、药明、迈瑞、百济、君实等 |
| 医疗 | 迈瑞、微创、乐普、鱼跃、凯利泰等 |
| 半导体/芯片 | 中芯、韦尔、卓胜微、北方华创等 |
| 银行 | 工商、建设、招商、平安、兴业等 |
| 保险 | 平安、人寿、太保、新华等 |
| 券商 | 中信、华泰、海通、国泰、招商等 |
| 科技 | 海康、科大、中兴、用友、恒生等 |
| AI | 科大、商汤、寒武纪、云从、海康等 |
| 汽车 | 比亚迪、上汽、长城、长安、吉利等 |
| 家电 | 美的、格力、海尔、老板、苏泊尔等 |
| 军工 | 中航、航发、中国卫星、中兵等 |
| 有色/煤炭/电力/化工/钢铁/地产 | 各对应行业龙头名称 |

匹配方式为对全市场股票名称做正则模糊匹配，而非行业分类。

#### 风格关键词

| 风格 | 市值门槛 | PE上限 | 说明 |
|------|---------|--------|------|
| 蓝筹 | ≥500亿 | ≤30 | 大盘低估值 |
| 白马 | ≥200亿 | ≤50 | 中大市值合理估值 |
| 成长 | ≥100亿 | ≤100 | 中等市值成长型 |
| 龙头 | ≥1000亿 | ≤200 | 超大市值 |

#### 技术面关键词

突破、金叉、放量、多头、上升趋势、底部、强势 —— 命中时额外 +3 分/项，最多 +9 分。

#### 基本面关键词

| 关键词 | 过滤条件 |
|--------|---------|
| 高ROE | ROE > 15% |
| 高增长 | 利润增速 > 20% |
| 低PE | PE < 30 |
| 低估值 | PE < 20 |
| 低负债 | 负债率 < 50% |
| 高分红 | 股息率 > 2% |

基本面条件作为**硬过滤**：不满足的直接排除。

### 筛选流程

```
用户输入 "高ROE低PE的白酒股"
         │
         ▼
┌─────────────────────┐
│ 关键词解析            │
│ 行业=白酒  风格=无    │
│ 基本面=高ROE、低PE    │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 全市场 ~5500只股票    │  → 排除 ST/退市/N/C
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 行业匹配              │  → 名称匹配"茅台|五粮|老窖|..."
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 基本面硬过滤           │  → ROE>15% 且 PE<30
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 市值排序，截取前24只    │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 并发评分（4线程，8秒超时）│
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 按总分降序排列         │
│ 输出前8只入池候选       │
└─────────────────────┘
```

## 前端展示

选股页面 (`ScreeningPage.tsx`) 包含：

- **搜索栏**：文本输入 + "筛选"按钮，支持 Enter 提交
- **示例查询**：6 个预设按钮（高ROE低PE白酒股、放量突破科技龙头 等）
- **大盘仪表盘**：4 张指标卡（大盘评分、入池数量、候选数量、筛选来源）
- **候选股票卡**：每只展示名称代码、5 维迷你环形图（绿/黄/红三色阈值）、入选原因、需确认问题
- **状态覆盖**：加载骨架屏、空状态提示、错误横幅

## 数据来源汇总

| 数据 | 来源 | 说明 |
|------|------|------|
| K线（日线） | Sina 前复权 / AKShare | 计算 MA5/MA20/MA60、技术指标 |
| PE/PB/市值 | Sina 实时 → AKShare → Tushare | 三源合并 |
| ROE/负债率/增速 | AKShare → Tushare | 深度财务 |
| 板块涨跌 | AKShare `stock_board_industry_name_em` | 行业热点 |
| 实时行情 | AKShare `stock_zh_a_spot_em` | 涨幅榜筛选 |
| 股票列表 | Sina 缓存（~5500只）→ AKShare | 增量同步，每日更新 |
| 大盘指数 | Sina / AKShare `stock_zh_index_daily` | 上证指数趋势判定 |
