# 选股实现逻辑

## 股票池概念

系统中有两层股票池：

### 全市场股票池（选股范围）

**A 股全部 ~5500 只个股**，自动排除 ST、退市、N（新股）、C（次新股）。

数据来源：
- Sina 实时行情定期同步到本地缓存（`~/.cache/stock_data/stock_basic.json`）
- 缓存按 Tushare 的 `stock_basic` 接口设计，含 `list_status`（L/D/P）、`list_date`、`delist_date`
- 支持增量更新：新上市、退市、改名均可识别
- 缓存过期时间 1 天，过期后自动同步

所有选股 API 均以此为基准池，再根据不同的入口场景施加过滤条件。

### 个人股票池（入选结果）

**经过五维模型评分后，总分 ≥ 70 分** 的股票进入个人观察池。这是选股的**结果**，不是选股的**范围**。

---

## 各场景的选股范围差异

| 场景 | 刷选范围 | 说明 |
|------|---------|------|
| **结构化选股** `GET /api/screening` | 涨幅前20全部评分 | 按 涨跌幅×成交额分位数 排序（兼顾强度与流动性），设计意图：捕捉当日市场中兼具弹性和流动性的强势股 |
| **自然语言选股** `POST /api/screening/natural` | 全市场 ~5500只 → 关键词过滤 | 按行业关键词、风格、估值条件过滤后，截取市值前24只评分 |
| **用户指定代码** `GET /api/screening?codes=xxx` | 用户指定的代码 | 直接对用户传入的代码列表评分 |
| **CLI 命令** `/选股` | 全市场 ~5500只 → 市值/PE过滤 | 按市值≥50亿、PE>0且≤200 过滤后评分 |

结构化选股的数据获取优先级：
1. 用户指定的代码列表（精确匹配）
2. 全市场实时涨幅榜（AKShare `stock_zh_a_spot_em`）
3. 以上均不可用时返回错误，不使用硬编码兜底池

所有场景均自动排除 ST、退市、N（新股）、C（次新股）标记的股票。

## 五维评分模型

总分 **100 分**，入池阈值 **≥ 70 分**。

### 1. 大盘环境 — 0~20 分

基于上证指数（000001）的 MA20/MA60 判定市场趋势：

| 条件 | 分类 | 加减分 |
|------|------|--------|
| 价格 > MA20 > MA60 | 进攻 | +5 |
| 价格在 MA20 与 MA60 之间 | 震荡 | 不变 |
| 价格 < MA20 且 < MA60 | 防守 | -3 |
| 指数涨跌幅 > +1% | — | +3 |
| 指数涨跌幅 < -1% | — | -3 |
| 成交量 > 20日均量 × 1.2 | — | +2 |

得分限制在 [0, 20] 区间。大盘环境评分对所有候选股票相同。

---

### 2. 政策/热点 — 0~20 分（按个股行业差异化）

调用 `ak.stock_board_industry_name_em()` 获取行业板块涨跌排行，通过 `stock_board_industry_cons_em` 获取各板块成分股，构建**股票→板块→涨跌幅**的映射：

- 板块涨幅 > 3% → **主线**
- 板块涨幅 > 1.5% → **支线**
- 其余 → **一日游**

每只个股按其所属板块的类别和涨跌幅差异化打分：

| 板块类别 | 得分范围 | 计算方式 |
|---------|---------|---------|
| 主线 | **15 ~ 20 分** | 15 + 涨幅/2 |
| 支线 | **10 ~ 15 分** | 10 + 涨幅/2 |
| 一日游 | **5 ~ 10 分** | 10 - abs(涨幅)/2 |
| 未匹配热门板块 | **8 分** | 固定 |

仅查询前 8 个热门板块的成分股（批量构建反向索引），未匹配到热门板块的股票默认为 8 分。

---

### 3. 基本面 — 0~25 分（行业相对PE）

基准分 **12.5 分**。PE 评分引入行业相对估值，先通过 `stock_individual_info_em` 获取个股行业，再匹配行业 PE 参考区间：

| 行业 | 低估线 | 高估线 |
|------|--------|--------|
| 银行 | < 4 | > 10 |
| 保险/券商 | < 8-10 | > 20-30 |
| 地产/钢铁/煤炭/建筑 | < 5-6 | > 15-20 |
| 有色/化工 | < 12-15 | > 35-40 |
| 白酒/消费/食品 | < 20 | > 45-50 |
| 家电/汽车 | < 10-12 | > 30-35 |
| 医药/医疗 | < 25 | > 60 |
| 新能源/光伏/电池 | < 15-20 | > 50-55 |
| 半导体/芯片/AI | < 30 | > 80 |
| 科技/军工/传媒 | < 25-30 | > 70-80 |

评分逻辑：PE ≤ 低估线 +4分，低估~高估之间 +2分，远超高估线(>2倍) -3分。无行业参考时使用通用阈值（PE<20 +3，PE<40 +1，PE>100 -3）。

| 指标 | 条件 | 加减分 | 含义 |
|------|------|--------|------|
| ROE（净资产收益率） | ROE > 15% | +4 | 盈利能力优秀 |
| ROE | 8% ≤ ROE ≤ 15% | +2 | 盈利能力良好 |
| 利润增速 | 增速 > 20% | +3 | 高增长 |
| 负债率 | < 50% | +3 | 低风险 |
| 负债率 | > 80% | -3 | 高杠杆 |

得分限制在 [0, 25] 区间。

财务数据来源于三源合并，每字段追踪来源和报告期：

1. **AKShare** `stock_financial_analysis_indicator` — 完整财务指标，含报告期
2. **Sina 实时接口** — PE/PB/市值/名称（实时快照，PE/PB 与 AKShare 交叉校验，差异 >20% 记录警告，实时数据优先）
3. **Tushare** `get_financial_indicators` — ROE/负债率/增速（深层财报，与 AKShare 交叉校验，差异 >15% 记录警告）

合并策略：
- 各源独立获取，不做硬优先级覆盖
- Sina PE/PB（实时）优先于 AKShare（静态报告期），其余字段以 AKShare 为主
- Tushare 补充 AKShare 缺失字段
- 返回 `_sources`（每字段来源标记）、`_report_dates`（报告期）、`_data_quality`（覆盖率和校验备注）

---

### 4. 技术面 — 0~25 分（趋势强度 + 放量位置）

基准分 **12.5 分**。

**均线排列：**
| 条件 | 加减分 | 含义 |
|------|--------|------|
| 价格 > MA5 > MA20 | +5 | 多头排列 |
| 价格 < MA5 < MA20 | -3 | 空头排列 |
| 价格 > MA60 | +3 | 站上60日线 |
| 价格 < MA60 | -1 | 低于60日线 |

**MA60 斜率（趋势强度）：** 对近20根K线的 MA60 做线性回归，计算斜率百分比：
| 条件 | 加减分 | 含义 |
|------|--------|------|
| 斜率 > 0.5%/期 | +2 | MA60上行，趋势强化 |
| 斜率 < -0.5%/期 | -2 | MA60下行，趋势弱化 |

**成交量 + 位置判断：** 放量时结合价格相对 MA60 的位置，区分是低位吸筹还是高位出货：
| 条件 | 加减分 | 含义 |
|------|--------|------|
| 放量且价格 < MA60 × 0.95 | +3 | 低位放量（吸筹迹象） |
| 放量且价格在 MA60 ±5% | +2 | 均线附近放量（方向选择） |
| 放量且价格 > MA60 × 1.2 | -2 | 高位放量（谨慎） |
| 放量但未达以上位置条件 | +1 | 放量 |

得分限制在 [0, 25] 区间。

---

### 5. 风险控制 — 0~10 分

固定为 **8 分**，未做动态计算。

## 评分流程

```
候选代码列表
     │
     ▼
┌─────────────────┐
│ 获取K线数据       │  AKShareClient.get_daily_kline(code)
│ 计算MA5/MA20/MA60│
└────────┬────────┘
         ▼
┌─────────────────┐
│ 获取财务数据       │  三源合并（AKShare → Sina → Tushare）
│ PE/PB/ROE/增速    │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 获取板块数据       │  AKShareClient.get_hot_sectors()
│ （大盘/政策维度）   │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 五维分别打分       │
│ 汇总总分          │
└────────┬────────┘
         ▼
┌─────────────────┐
│ total >= 70 ?    │─── 是 ──→ 纳入个人股票池
│                  │─── 否 ──→ 过滤掉
└─────────────────┘
```

结构化选股对涨幅榜前10全部评分，自然语言选股对过滤后的全部股票并发评分（上限100只防超时），总分 ≥70 的全部返回，不设数量上限。

## 自然语言选股

### 关键词解析

系统将用户输入的自然语言拆解为四个维度：

#### 行业关键词（18个行业）

| 行业 | 匹配的品牌/股票名模式 |
|------|---------------------|
| 白酒 | 茅台、五粮、老窖、汾酒、古井、洋河等 |
| 消费 | 伊利、蒙牛、海天、金龙鱼、双汇等 |
| 新能源 | 宁德、比亚迪、阳光电源、隆基、通威等 |
| 电池 | 宁德、亿纬、国轩、欣旺达、德赛等 |
| 光伏 | 隆基、通威、阳光、晶澳、天合、中环等 |
| 医药 | 恒瑞、药明、迈瑞、百济、君实等 |
| 医疗 | 迈瑞、微创、乐普、鱼跃、凯利泰等 |
| 半导体/芯片 | 中芯、韦尔、卓胜微、北方华创等 |
| 银行 | 工商、建设、招商、平安、兴业等 |
| 保险 | 平安、人寿、太保、新华等 |
| 券商 | 中信、华泰、海通、国泰、招商等 |
| 科技 | 海康、科大、中兴、用友、恒生等 |
| AI | 科大、商汤、寒武纪、云从、海康等 |
| 汽车 | 比亚迪、上汽、长城、长安、吉利等 |
| 家电 | 美的、格力、海尔、老板、苏泊尔等 |
| 军工 | 中航、航发、中国卫星、中兵等 |
| 有色/煤炭/电力/化工/钢铁/地产 | 各对应行业龙头名称 |

匹配方式：优先通过 AKShare `stock_board_industry_cons_em` 获取标准行业板块成分股（如"白酒"→"酿酒行业"），名称正则模糊匹配作为兜底。

#### 风格关键词

| 风格 | 市值门槛 | PE上限 | 说明 |
|------|---------|--------|------|
| 蓝筹 | ≥500亿 | ≤30 | 大盘低估值 |
| 白马 | ≥200亿 | ≤50 | 中大市值合理估值 |
| 成长 | ≥100亿 | ≤100 | 中等市值成长型 |
| 龙头 | ≥1000亿 | ≤200 | 超大市值 |

#### 技术面关键词

突破、金叉、放量、多头、上升趋势、底部、强势 —— 命中时额外 +3 分/项，最多 +9 分。

#### 基本面关键词

| 关键词 | 过滤条件 |
|--------|---------|
| 高ROE | ROE > 15% |
| 高增长 | 利润增速 > 20% |
| 低PE | PE < 30 |
| 低估值 | PE < 20 |
| 低负债 | 负债率 < 50% |
| 高分红 | 股息率 > 2% |

基本面条件作为**硬过滤**：不满足的直接排除。

### 筛选流程

```
用户输入 "高ROE低PE的白酒股"
         │
         ▼
┌─────────────────────┐
│ 关键词解析            │
│ 行业=白酒  风格=无    │
│ 基本面=高ROE、低PE    │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 全市场 ~5500只股票    │  → 排除 ST/退市/N/C
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 行业匹配              │  → 名称匹配"茅台|五粮|老窖|..."
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 基本面硬过滤           │  → ROE>15% 且 PE<30
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 市值排序，截取前100只   │  ← 防超时上限
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 并发评分（4线程，8秒超时）│
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 按总分降序排列         │
│ ≥70分全部返回，不设上限 │
└─────────────────────┘
```

## 前端展示

选股页面 (`ScreeningPage.tsx`) 包含：

- **搜索栏**：文本输入 + "筛选"按钮，支持 Enter 提交
- **示例查询**：6 个预设按钮（高ROE低PE白酒股、放量突破科技龙头 等）
- **大盘仪表盘**：4 张指标卡（大盘评分、入池数量、候选数量、筛选来源）
- **候选股票卡**：每只展示名称代码、5 维迷你环形图（绿/黄/红三色阈值）、入选原因、需确认问题
- **状态覆盖**：加载骨架屏、空状态提示、错误横幅
